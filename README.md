# Software Foundation Documents Standard

_Version 1.0_

This is a three-document-suite standard designed to serve as the project's constitutional documentation.

The main characteristics of this standard includes:
- Living Source of Truth - These Foundation Documents are the living, evolving primary Source of Truth for the entire project.
- AI-first documentation - Designed for AI agents to interact through time, with human-in-the-loop responsibility.
- Hierachical documentation - the the Foundation Documents serve as the project's top-most documentation that nothing can violate them. The per-version implementation plan and per-service specificiation may be made as see fit but never contradict these Foundation Documents.
- Permanance by design - All decision are recorded immutably. Key reference points must be cited throughout, downwards towardds lesser documentations. All reference points are fixed and can never be changed once published. Made with legal-like specificity and permanance. 

It is hard for human writers to manually edit and maintain these documents by hand, especially for untrained documentation editors. Strongly recommend to delegate the editing and maintaining task to frontier-class AI agents.

## Standard Handbook

This standard comes in 2 variants:
1. The full standard (```software-foundation-documents-standard-v1_0.md```) - This is the complete, full-details, official standard for the SFD.
2. The agent guide (```sfd-agent-guide-v1_0.md```) - This is the distilled version of the full standard meant to be given to AI agents for reference.
